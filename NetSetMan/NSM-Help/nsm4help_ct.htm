基礎
簡介

NetSetMan 是一個網絡設置管理軟體，它可以輕鬆地切換6個不同的、直覺式的配置檔包括：

  • IP 位址 (IPv4 和 IPv6 !)
  • 預設閘道 (包括 Metric)
  • DNS 伺服器
  • WINS 伺服器
  • WiFi 管理
  • 電腦名稱
  • 工作群組
  • 預設印表機
  • 網絡磁碟
  • 網路卡狀態
  • 連線 (RAS / DUN / PPP / VPN)
  • DNS 尾碼
  • 路由表
  • SMTP 伺服器
  • Hosts 檔案內容
  • 系統設置
  • MAC 位址
  • 腳本 (BAT, VS, JS, ...)
  • 執行程式
 專業版額外功能：
  • 允許應用在工作上
  • 配置檔數量無上限
  • 網路網域
  • 完整的代理伺服器設定和
    瀏覽器首頁
    (Internet Explorer, Firefox, Opera)

NetSetMan 為您提供 Windows 至今仍不支援的功能：
一個功能強大、易於使用的界面來管理所有的網絡設定，一目瞭然。
一試上癮，再也不能沒有它。

所有的設定和更改都自動保存在一個獨立的檔案 (settings.ini)。備份、轉移或升級到另一個版本都沒有相容性問題。

一個按鍵即可套用已儲存的配置檔甚至全自動套用！

基本資訊
  • 配置檔
    每個配置檔代表一組針對特定位置或場景所需要的設置。通過使用子配置檔，多個網絡介面卡可以使用同一個配置檔來配置。透過雙擊頁籤標題可以替配置檔重新命名。
  • 手動配置
    您需要自己手動建立所需的配置檔，對於一個陌生的地點要去猜測正確的設置為何是不可能達成的。此應用程式的主要目的是對於您通常需手動輸入去改變設置的部分，可以自動地來完成。
  • 設定選擇
    讀取所有當前的設定成一個配置檔可以給您一個起點，但您仍然需要決定當該配置檔被應用時所要套用的設定。強烈建議把要套用的部分限縮於那些實際上會同時改變的設定，例如，如果電腦名稱並不需要改變，就沒有必要套用它。
  • 設定設置
    要編輯一個設定前，首先要把選取框打勾，然後直接在主界面中輸入需要的設定或打開設定對話框輸入特定的設定。
  • 配置檔套用
    預設情況下，沒有配置檔會被自動套用，也就是說如果不是您想要的話，系統中沒有設定會改變。雖然如此，您可以使用自動切換功能，自動的套用一定的配置檔根據自由設定的條件。
  • 系統列圖示存取
    建議開啟 保持 NetSetMan 最小化於系統列 ，然後更改系統列圖示為總是顯示，這樣，您就可以透過點擊該圖示並選擇配置檔來快速套用。此外，您可以透過懸停您的滑鼠游標在圖示上來獲取您當前連線 IP 的有用資訊。
配置檔

頁籤代表了可用的配置檔，您可以儲存6組不同的配置在這些分頁裡。

如果您想在同一個配置檔內改變兩張不同網絡卡的設定，您可以使用子配置檔功能。點擊下拉式選單旁邊的 "+" 按鈕，一個數量為兩個的子配置檔（當前的和一個新的、空白的）的選單列將出現，標題將會自動填上在子配置檔內被選取的網絡卡名稱而且是不能更改的。您可以依您的喜好建立多個子配置檔，所有的子配置檔都將被依序套用當配置檔套用時，您可以透過滑鼠拖曳重新安排套用順序。

所有的改變都會被自動儲存，不會意外遺失。

透過點擊右鍵並選擇 "重新命名" 或直接雙擊可以重新命名配置檔。

您可以使用在選單列的配置檔管理來更方便管理您的所有配置檔。它允許透過滑鼠游標或一些常用的鍵盤快速鍵（方向鍵為瀏覽、F2 為重新命名...等）編輯配置檔、子配置檔和群組(僅在 NetSetMan Pro)。當使用拖放重新安排項目時，根據不同的項目，由於情況的不同會發現一些不可避免的限制：配置檔不能刪除、子配置檔不能重新命名、配置檔可以互換，但不能移動。選取框的勾選與否代表該配置檔會不會顯示在系統列選單和精簡模式中。

網絡卡(NICs)

所有可用的網絡卡依名稱和類型來顯示。在當前的(子)配置檔選擇您想管理的網路卡。

如果需要的網路卡在列表中找不到（例如: 已停用、虛擬的），您可以通過點擊重新整理按鈕快速更新列表或者選擇管理網絡介面卡來管理列表。然後您可以看到一個完整的可用網路卡清單，您可以在其中找尋您要的網路卡。

註：所有在該清單內的網路卡都會出現在系統列訊息中。取消勾選不需要的網路卡可以讓它們在清單和系統列訊息中被刪除。

如果複製了設定檔（settings.ini）到另一個系統，NetSetMan 會嘗試自動重新分配配置檔中的網路卡，然而，不過只有當新系統有相同名稱的網路卡存在時才會有效(例如截圖中的 "LAN")。

設定操作

載入目前設定

要載入目前設定到選擇的配置檔中只需點擊輸入區域旁的 符號。
若要取得所有目前的設定，請在配置檔名稱點擊右鍵並選擇 "載入目前設定"。

清除設定

要清除某個設定的配置，請右鍵點擊它並選擇"清除"。

如果要清除某個 IP 輸入欄位，只要雙擊它的標題即可。

要清除目前(子)配置檔的所有設定，請在配置檔名稱點擊右鍵並選擇 "清除設定"。

複製與貼上設定

如果您要在不同的配置檔間移轉設定，您可以複製並貼上整個配置檔：透過右鍵點擊第一個來源配置檔，選擇 "複製設定"；然後右鍵點擊新配置檔，並選擇 "貼上設定"。

要複製和貼上某一特定的設定，只需右鍵點擊並選擇"複製"或"貼上"。

套用配置檔

要套用當前配置檔的所有設定，請點擊"套用"；如果程式運行在最小化時，可以從系統列選單中選擇的配置文件 。
注意：只有選取框被打勾的設定才會被套用。

套用對話框包含了有關當前套用狀態的訊息。
您可以看到哪些設定將要被套用和過程是否成功。您可以點擊 "取消" 按鈕取消套用進程。

所有的更改都會即時生效。然而，在某些情況下，某些設定需要一段時間才能完全套用成功，或者需要重新啟動(尤其是"電腦名稱"和"工作群組/網域")。

詳細資訊部分提供了該配置檔的所有配置設定的概述，要查看更多的資訊和錯誤訊息，請打開日誌部分。

對於常見的錯誤和問題，您可以參閱 FAQ 找尋解決方法 。

精簡模式和完整模式

除了傳統的 "完整模式" 界面讓您可以改變所有配置檔的設定外，還有一個額外的 "精簡模式" 可供使用，您可以隨時透過點擊在工具列右方的按鈕來切換模式。

一旦所有配置檔的設定都完成後，精簡模式是理想的操作方式，它提供一個配置檔所有設定的精簡概述，而且不必擔心設定被不小心修改到。

如果您使用 NSM 管理（"更改配置檔設定" 未選取），精簡模式將自動被啟用來防止用戶改變設定。

系統列選單和資訊

當 NetSetMan 運行中時，您會在系統列發現它的圖示（時鐘旁邊）。
透過圖示您可以快速套用一個配置檔，只需右鍵點擊圖示打開系統列選單然後選擇要套用的配置檔。

如果您想排除某些配置檔在系統列出現，您可以個別地隱藏它們。要做到這一點請先在主視窗中選擇配置檔，右鍵點擊該分頁標籤並點擊 "在系統列選單和精簡模式中顯示"。文字旁邊的小符號表示此配置檔能不能在系統列被啟用，這個選項在精簡模式也會一體適用。配置檔的顯示隱藏也可以在配置檔管理中設定 。

如果您把游標懸停在 NetSetMan 系統列圖示上，系統列訊息會被顯示，它包含了當前系統使用的 IP 訊息。由於這些數據是即時產生的，所以顯示時會有短暫的遲延，為了減少這種延遲，您可以考慮在選項裡關閉額外的訊息，例如公共 IP、MAC 位址等。

功能
總覽

NetSetMan 介面設計的方式，可以讓您對於配置檔內所有配置的設定有一個直接的總覽。像基本的 IP 設定、WiFi 和 預設印表機這些最重要的功能可以直接在主界面上進行配置，至於其他更複雜的設定，在這裡只能被套用，並會顯示在他們的當前配置的簡要概述。想要修改的話，首先要勾選它們，然後點擊[...]打開根據不同項目所呈現的不同對話框。

以下各節會給您的基本的說明和有用的提示來使用所有功能。

介面卡狀態

使用介面卡狀態按鈕來啟用、停用或重新啟用所選擇的網絡介面卡，預設的顏色會變成一個和狀態相關的顏色；另外，您也可以在工具提示看到配置的狀態。

如果您要啟用一個介面卡和停用一個或所有其他的介面卡，您需要為每個介面卡建立子配置檔，您可以手動或使用的介面卡狀態選單底部的選單項目做到這一點，它會替包含在介面卡清單中的每一個網路介面卡自動建立一個子配置檔，並設定其相應的狀態。再次提醒，您可以透過網絡介面卡管理來決定哪些介面卡會被顯示在清單之中。

進階 TCP/IP 設定

如果您需要同時存取數個網絡，您可以針對每一個網路介面指定多組 IP、閘道、DNS 和 WINS 位址。如果想這樣做，請從主選單中點擊 "IP +" 按鈕或選擇"檔案 > 進階 TCP/IP 設定"。

在這裡，您還可以管理更進階的設定，例如：路由表、DNS 尾碼、IPConfig 和 NetBIOS 。

  • 透過勾選標題複選框套用該群組
  • 增加一個新的值
  • 刪除選取的值
  • 清除整個清單
  • 從系統獲取此設置所有值(在主視窗中點擊 符號只有讀取的第一個值)
  • 透過拖放重新排序
連線

使用此功能可以連接或中斷任何Windows的連線，例如：撥號網絡，移動寬頻或 VPN。相容的連線會在視窗中和網絡連線對話框中正常的網絡介面卡一起被列出來。以技術上來講，所有和 "RASDIAL" 相容的連線都有支援。

您可以選擇透過勾選複選框來斷開所有當前建立的連線，或者個別地 連接/中斷 某些連線。

如果你的連線因為某些原因並不存在清單之中，你可以手動的增加它透過點選選項清單中的 "新增..." 並輸入它在 Windows 中顯示的名稱。

NetSetMan 也支援許多第三方的 VPN 用戶端，例如：OpenVPN、Shrew 和 Cisco。如果你的 VPN 用戶端不相容的話，NetSetMan 將會在選項清單中列出它的 VPN 配置檔。

主機檔

在非常特殊的情況下，可能有必要或有益的需手動指定 IP 位址到主機名稱或網域名。請注意，NetSetMan 將只會修改自己管理的主機項目，其他已存在 hosts 檔裡的項目將保留不變。

網絡磁碟

透過 NetSetMan 您可以連線和中斷無限數量的網絡磁碟機。
請點擊在視窗底部的 "[顯示提示]" 並閱讀使用說明。

當一個網絡磁碟機功能已選取的配置檔被套用時，NetSetMan 會將已連線的和將要連線的網路磁碟機列表進行比對，所有同時存在(相同的磁碟機代號和路徑位置)於兩份列表中的網路磁碟機將保持不變；不存在的磁碟將被中斷；最後再連線新增的磁碟。

腳本和程式

NetSetMan 為您提供一個舒適的方式來管理您的腳本檔案，然而，它沒有自訂的腳本語言，這意味著您只能用您的作業系統支援的腳本語言來執行。常見的腳本語言有 VBS、JS 甚至於單純的 DOS 批次檔，您可以使用您喜歡的搜索引擎找到更多有關腳本的訊息和它們可以支援的功能。

打開一個現有的腳本檔案或新建一個以供編輯。
您可以在所有其他設定被更改前和更改後各執行一個腳本。
如果在執行腳本和更改網絡設定中間需要一些遲延，您可以在底部設定它。

執行程式

NetSetMan 提供一個簡便的方法伴隨著配置檔來執行外部的程式。
程式可以以一般使用者身分(預設 UAC 開啟)或者以系統管理員身分(不會觸發 UAC 提醒)啟動，需要的話還可以添加啟動參數。
程式可以啟動為一般視窗大小、最大化或最小化。

正在執行中的程式可以根據程式名稱(不需輸入程式位置)或確切的檔案路徑來被找到和中止執行。

每一種動作都可以設定分別的配置檔套用前(負值)或套用後(正值)的延遲時間。

SMTP / 電子郵件

在某些情況下，SMTP 伺服器發送郵件的 IP 地址取決於您所處的網絡。在內部網絡時，您可能需要像 10.0.0.1 這樣的本地 IP 位址；而當在外部網路時，您可能需要一個全球性的 IP 位址來存取它。

相對於試圖完全相容於許多不同的電子郵件客戶端，並處理不同的用戶配置檔，NetSetMan 透過一個簡單的方法解決這個問題。設定 "nsm.smtp" 作為您的電子郵件客戶端的 SMTP 位址，並根據配置檔設定 IP 位址，當套用該配置檔，這兩個值都將透過使用 hosts 文件連接，這樣您就不需要對於您的電子郵件客戶端在做任何其他的更改。

系統設定

除了通常的網絡設定之外，有時您可能也想改變一些系統設定取決於您的位置和情況。例如：在學校，您可能想要音量保持靜音、更換壁紙、套用平衡電源計劃。

您也可以使用系統設定窗口來快速更改一些設定，否則將花費更多的時間在尋找該設定在 Windows 中的位置。只需點擊 按鈕直接套用特定的設定。

支援的設定包括：顯示模式、桌面背景、桌面顏色、螢幕保護程式、佈景主題，喇叭和麥克風的音量、音效配置，防火牆(包括異常和通知)、電源計劃、時區和 MAC 位址。

WiFi

NetSetMan 配置檔可用於連接到一個已存在的 WiFi 配置檔。要做到這一點必須先在網路卡下拉式選單中的選擇一個 WiFi 介面，然後再從 WiFi 下拉式選單中選擇一個 WiFi 配置檔。如果您想要中斷您的 WiFi 連線，選擇一個空的配置檔即可。

除了基本的 WiFi 切換， NetSetMan 提供了豐富的 WiFi 管理。點擊 WiFi 圖示或使用工具列(工具 > NSM WiFi 管理)來存取它，您會發現一個現在可用的 WiFi 列表。黃色的背景表示一個具有相同 SSID 的配置檔已經存在；綠色的背景表示是目前的連線。您可以連線到一個網絡透過雙擊它或使用 "連線" 按鍵，如果這個連線的配置檔不存在，您可以選擇先建立它。設定(驗證、加密、...)將自動根據識別的網絡預先設置，然而您可能需要調整它當有些原因造成自動識別不能正常工作的情況下。

NetSetMan 與 Windows 作業系統的管理無線網路連結，這意味著，您存取的配置檔和 Windows 或其他無線網絡軟體已建立的是相同的，您可以新建、修改或刪除無線配置檔。

依 MAC 位址連線

Windows 通常依據 SSID 來連線到無線網路。如果多重的基地台擁有一樣的 SSID，Windows 會自動歸類並連線到訊號最強的那一個。然而，有時候你想要指定連線到某個特定的基地台，這可以透過 NetSetMan 來實現!

NSM 的 WiFi 管理不會自動歸類而是顯示所有可以連線的網路，因此你可以建立你真正想要的連線。滑鼠懸停透過提示來顯示每個連線的額外資訊，例如：MAC 位址，開啟右鍵選單就可以輕鬆地複製 MAC 位址或 SSID。

如果你想要在套用 NetSetMan 配置檔時連線到一個特定的無線網路，你可以透過加入它的 MAC 位址到 WiFi 選項中來完成。點選 "新增..." 然後輸入你選擇的 MAC 位址，當套用配置檔時，NetSetMan 將連線到相對應的基地台。注意：你仍然需要擁有一個適合的 WiFi 配置檔存在你的系統裡。

需求

NetSetMan 使用 Windows 作業系統的 WiFi API，這帶來了很大的優勢，沒有新的驅動程式需要安裝(因為可能導致一些衝突)。雖然在底層的 NetSetMan WiFi 功能只能在已經支持 WiFi 的 Windows 使用，但好消息是現今大部分的版本都是如此，如果您使用的是 Vista 或更高版本，將不會有相容性問題；XP 則必須要是 SP2 或 SP3 版本。

請確認 WLAN AutoConfig(Wlansvc)(Vista和更高版本)服務或 Wireless Zero Configuration(WZCSVC)(XP)服務是啟動的。

如果您的無線網卡附有一個製造商客製化的 WiFi 管理軟體，通常大多數情況下它會阻止 Windows 內建的管理服務存取 WiFi，您需要找到客製化管理軟體內的選項(如果有的話)來切回到用 Windows 管理。
因為有很多不同的製造商擁有各自的客製化軟件，所以沒有通用的教學方法來達成上述目標，下面是一個範例：如果您使用的是 AVM FRITZ!WLAN USB 網卡，點擊系統列軟體圖示然後選擇 "Windows WLAN"。

進階
自動切換

點擊系統列選單可以很容易的套用一個配置檔，但在某些情況下，根據自定義的條件完全自動啟動配置檔的話可能會讓事情更容易些，整合的自動切換功能正是應此而生的。

開啟自動切換之後，下列的條件可以於個別的配置檔中被設定：

周圍的 WiFi 網絡
  • 無線網絡(SSID 或 MAC 位址) 進入/超出 範圍
  • 無線網絡信號強度 低於/超過 一定值
網絡位置
  • Windows NLM：自動由 Windows(Vista 或更高版本)產生的網絡配置檔
  • 閘道器 MAC：DHCP (自動分配IP)閘道器的指紋
  • Ping：存在網絡中的某個伺服器或系統
網路卡狀態
  • 本機網絡網路卡(連線/中斷) 的連線狀態
區間
  • 星期幾和時間區間(例如：星期六和星期天 8:00 - 18:00)

雖然對於每一個配置檔可以設定的條件數量沒有技術上的限制，不過請儘可能保持簡單。在大多數情況下，以一個或兩個條件來實現期望的結果是絕對足夠的。

您可以指定無論是全部或至少有一個條件必須符合，就可以套用該配置檔。

註：當多個配置檔的條件在同一時間都符合時，將由低 ID 的配置檔有優先權。

在預設情況下，切換進程初始化完成後，在畫面右下角會有一個可以自訂的延遲倒數計時，讓您可以選擇性地中止切換進程。另一個選擇是，本機的 Windows 通知可以用來顯示訊息或者配置檔可以在背景運作切換且不會顯示任何訊息。在第二種狀況下，工作列圖式會在切換過程中以動畫顯示，並且根據切換的結果即時的改變它的顏色來提示您自動切換的進程。

為了節省系統資源， NetSetMan 不會比較您當前的設定和將要套用的配置檔，這意味著，如果您手動更改任何設定，NetSetMan 不會注意到此變化。

自動切換在 XP 中的一些限制：Windows NLM 無法使用、周遭的 WiF i網絡變化可能會有明顯的延遲。

註解

您可以為每個配置檔建立簡短的說明來幫助您記住或註明該配置檔的用途。如果一個配置檔含有註解，它在選單列的圖示會以彩色的方式出現，該註解在完整模式將以圖示提示的方式顯示；在精簡模式中則顯示在配置檔設定的頂部。

NSM 服務 / 受限制用戶帳戶

NetSetMan 需要管理員權限才能更改網絡和系統設定為您服務，通常執行這樣的行為在受限制用戶帳戶是不可能的。較新的 Windows 版本，甚至使用 UAC (用戶帳戶控制) 限制管理員賬戶，在每個程式啟動前顯示訊息。

為了解決這個問題， NetSetMan 內含一個叫做 NSM 服務的特殊應用，它能提供主程式所需的權限。該服務會被自動安裝，但也可以手動在選項(系統權限 > 啟用/停用）中做 安裝/移除 的動作。

已經安裝的服務可以在 Windows 服務對話框(Services.msc)找到和調整它。此外，它也可以使用命令列 安裝/移除 或 啟動/停止：
nsmservice.exe /install /silent
nsmservice.exe /uninstall /silent
net start nsmService
net stop nsmService

如果遇到"無效的簽名"錯誤，請確認該執行檔並沒有被修改，還有系統群組原則 "Turn off Automatic Root Certificates Update" 並未啟用。

如果您在一個受限制用戶帳戶裡執行 NetSetMan 且 NSM 服務沒有啟用的話，您將會收到一條訊息說明這個問題。作為替代，您可以在本地帳戶輸入管理員憑據 ，NetSetMan 將會將它加密儲存在設定檔中，並用它來啟動自己的所需的權限 。請注意，在這種情況下，應用程式將在其憑證使用用戶的背景中執行。

查看 FAQ 以瞭解有關該功能的一些其他注意事項。

登入前配置檔預切換

一個非常獨特而強大的功能是可以在登入畫面上套用您的配置檔。這使得系統得以準備您的帳戶所需要的正確的網絡設置透過在登入之前套用 NetSetMan 的配置檔。

要使用此功能， NSM服務 必須安裝並勾選"在 Windows 登入畫面啟用配置檔切換"(選項 > 系統權限)。

請注意，此功能只允許改變用戶之間共享的設定，個別用戶特定的設定，並不在此列。

NSM 管理

如果您設定的 NetSetMan 正在被別的使用者使用時，您可能想要指定哪些功能是允許其他使用者執行的。這可以使用 NSM 管理員(參數設定 > 管理員)來完成。

當您要啟用這個模式，必須先輸入一個大於或等於6個字元的密碼，只有當密碼通過驗證後，您才可以改變 NSM 管理設定，或執行那些被保護的功能(啟動 NetSetMan、套用配置檔、更改配置檔設定、更改程式設定)。

您甚至可以鎖定某些配置 ，讓用戶不能修改、但是仍然能夠套用它們。或者您也可以直接限制訪問某些設置 例如"工作群組和網域"，但同時允許配置和使用所有其他設置。

密碼的校驗值會被寫入在登錄檔，請確保一般使用者沒有 HKLM\SOFTWARE\NetSetMan (x64：HKLM\SOFTWARE\Wow6432Node\NetSetMan) 的寫入權限。如果您忘記了密碼，您將需要手動刪除該註冊碼，如果您的設定檔是加密的，沒有密碼的人就無法查看或編輯它。

命令列套用

在某些特殊情況，您可能需要在不使用 NetSetMan 圖形介面下更改您的配置檔：使用快捷方式更改配置檔、在啟動時套用配置檔、將它包含在一個複雜的腳本內......等等。

以下為可用的參數：

-a # 套用編號為 # 的配置檔
-as # 在安靜模式下套用配置檔(如果沒有錯誤發生，套用視窗將自動關閉)
-h / -hide 最小化執行 NetSetMan
-f / -full 在完整模式下執行 NetSetMan(預設值)
-c / -compact 在精簡模式下執行 NetSetMan
-m / -multi 執行第二個 NetSetMan 程式。 小心：所有同時執行的 NetsetMan 都寫入相同的設定檔！

一個快捷方式啟動配置檔 #0 在安靜模式下的指令為：
"C:\Program Files (x86)\NetSetMan\netsetman.exe" -as 0

專業版額外功能
配置檔群組

透過 NetSetMan Pro，您可以建立一個無限數量的配置檔群組，每個群組包含6個配置檔。

要新增、刪除、重新編排或重新命名配置檔群組，請使用工具列中的配置檔管理。

要瀏覽所有可用的配置檔群組，請從工具列的下拉式選單選擇任何一個群組。

如果您有多個群組，您的配置檔會以群組方式顯示在系統列選單中。

瀏覽器 / 代理伺服器

透過 NetSetMan Pro，您可以管理 Windows/Internet Explorer、Firefox 和 Opera(<= v12) 的代理伺服器。Chrome 和更高版本的 Opera 不支援自訂的代理伺服器設定，而是自動使用系統設定。

所有的設定是為了模仿原介面，來允許做到原本瀏覽器直接也可以完成的相同設定；另外，您也可以更改首頁，如果您在Firefox或Opera有使用不同的使用者設定檔，您也可以選擇使用者設定檔名稱。

請注意 Firefox 在套用新的設定之前，必須先關閉。
如果您的 Firefox 有多個使用者設定檔而且資料夾位置有改變(例如：因為安裝新系統...等)，您將需要在列表中重新選擇使用者設定來更新您的設定檔。

網域

要加入網域，請輸入名稱、您的用戶名稱和密碼。您的用戶名稱和密碼將被加密保存在 ini 檔案中。

通常單純輸入用戶名稱就已經夠了，但是，如果您遇到錯誤，您可以嘗試輸入下列兩個常見標準的其中一種作為您的用戶名稱：
a) NETBIOS: DOMAIN\user
b) UPN: user@domain.local

在 NT 網域，用戶在同一時間只能是其中一個網域的成員。這意味著，加入另一個網域或工作群組中時，前一個網域成員的身份就會被丟失，直到您的系統再重新加入。

NetSetMan 只能幫您加入那些您的網域用戶帳戶已經擁有權限的網域。
該用戶帳戶的權限被稱為"添加工作站到網域"，且必須在您的帳戶中被啟用，如果您沒有這個權限，請諮詢您的網絡管理員申請。
