Principes de Base
Description

NetSetMan est un gestionnaire de paramètres de réseau qui permet de basculer facilement entre 6 profils différents, visuellement structurés pour votre confort, incluant :

  • Adresses IP (IPv4 et IPv6 !)
  • Passerelles (incl. Metric)
  • Serveurs DNS
  • Serveurs WINS
  • Gestion du WiFi
  • Nom de l'ordinateur
  • Workgroup
  • Imprimante par défaut
  • Lecteurs réseau
  • Etat des interfaces réseaux
  • Connexions (RAS / DUN / PPP / VPN)
  • Suffixe DNS
  • Table de routage
  • Serveur SMTP
  • Entrées du fichier hosts
  • Paramètres système
  • Adresse MAC
  • Scripts (BAT, VS, JS, ...)
  • Exécution du programme
En complément dans la version Pro uniquement:
  • Peut être utilisé au travail
  • Nombre illimité de profils
  • Réseau-domaine
  • Gestion complète des paramètres du proxy et
    Page d'accueil du navigateur
    (Internet Explorer, Firefox, Opera)

NetSetMan vous offre tout ce qui vous manquait dans Windows jusqu'à présent :
Une interface puissante et facile à utiliser pour gérer tous les paramètres de votre réseau en un clin d'œil.
L'essayer : c'est l'adopter !!!

Tous les réglages et modifications sont automatiquement enregistrées dans un fichier séparé (settings.ini) facilitant ainsi backup, transferts ou mise à jour vers une autre version majeur.

Activer un profil stocké en un seul clic ou automatiquement !

Renseignements de base
  • Profils
    Chaque profil représente un ensemble de paramètres qui sont nécessaires pour vous connecter en fonction de votre localisation ou d'un scénario. En utilisant les sous-profils, plusieurs interfaces réseaux peuvent être configurées avec un seul profil. Renommer un profil en double-cliquant sur son onglet.
  • Configuration manuelle
    Vous devez configurer manuellement le profil requis. Il n'est pas possible de deviner les bons paramètres pour une localisation inconnue. Le but principal de l'application est d'automatiser les changements de réglages que vous devez généralement faire manuellement.
  • Sélection des paramètres
    La lecture de tous les paramètres dans un profil peut vous donner un point de départ, mais vous avez encore besoin de décider quels paramètres s'appliquent lors de l'activation de ce profil. Il est fortement recommandé de ne choisir que les paramètres qui doivent réellement être changés. Par exemple, il n'est pas nécessaire d'appliquer un nom d'ordinateur si ce paramètre n'est jamais changée.
  • Configuration des paramètres
    Pour configurer un paramètre cocher la case, puis entrez soit la configuration souhaitée directement dans l'interface principale soit ouvrir la boîte de dialogue de configuration pour ce paramètre particulier.
  • L'activation d'un profil
    Par défaut, aucun profil n'est automatiquement activé. c-a-d, sans action de votre part, aucun changement ne sera réalisé sur votre système. Cela dit, vous pouvez utiliser la fonction AutoSwitch pour activer certains profils automatiquement en fonction de conditions entièrement personnalisables.
  • Accès aux icônes de la barre de notification
    Il est recommandé de garder NetSetMan minimisé dans la barre de notification et de conserver son icône toujours visible. De cette façon, vous pouvez rapidement activer un profil en cliquant sur l'icône et en sélectionnant un profil. En outre, vous aurez un aperçu de vos adresses IP de connexions actuels en survolant l'icône avec le curseur de votre souris.
Profils

Les onglets représentent les profils disponibles. Vous pouvez enregistrer 6 différents ensembles de paramètres dans ces onglets.

Si vous souhaitez modifier les paramètres de deux interfaces réseaux différentes lors de l'activation d'un seul profil, vous pouvez utiliser la fonction sous-profil. Cliquez sur le bouton "+" à côté de la liste déroulante et une nouvelle barre de menu apparaîtra contenant deux sous-profils (l'actuel et un autre vide). La légende sera automatiquement le nom de l'interface réseau qui est sélectionnée dans ce sous-profil et ne peut pas être changé. Vous pouvez créer autant de sous-profils que vous le souhaitez. Ils seront activés l'un après l'autre lorsque le profil sera activé. Vous pouvez modifier l'ordre par glisser-déposer.

Toutes les modifications sont automatiquement enregistrées afin que rien ne se perdre accidentellement.

Vous pouvez renommer un profil avec un clic-droit dessus et en choisissant "Renommer" ou simplement en double-cliquant sur son onglet légende.

Pour gérer plus facilement tous vos profils, vous pouvez utiliser le menu Gestion des profils que vous trouverez dans la barre de menu. Il permet la modification des profils, sous-profil et les groupes (uniquement avec NetSetMan Pro) avec la souris ou avec des raccourcis clavier standards (touches fléchées pour la navigation, F2 pour renommer, etc). Lors de l'utilisation du glisser-déposer pour réorganiser les éléments, vous remarquerez certaines restrictions liées à des contraintes spécifiques : Les profils ne peuvent pas être supprimés, les MultiProfiles ne peuvent pas être renommés, les profils peuvent être échangés, mais pas bougés. Les cases à cocher indiquent si le profil en fonction sera visible dans la barre de notification et en mode compact.

Interface réseau (NIC)

Toutes les interfaces réseaux disponibles sont affichées par leur nom et leur type. Sélectionnez l'interface que vous souhaitez gérer dans le (sous) profil actuel.

Si une interface n'est pas disponible dans la liste (par exemple, parce qu'elle est désactivée ou virtuelle), vous pouvez actualiser la liste en cliquant sur ​​le bouton Actualiser ou en sélectionnant le menu Gestion des interfaces réseaux. Vous y trouverez une liste complète de toutes les interfaces réseaux disponibles. Vérifiez que toutes les interfaces réseaux que vous voulez avoir sont présentes dans la liste.

Remarque: Toutes les interfaces réseaux qui sont incluses dans cette liste apparaîtront également dans les informations de la zone de notification. Décochez les interfaces que vous voulez supprimer.

Si vous copiez le fichier de paramètres (settings.ini) sur un autre système, NetSetMan va essayer de réaffecter les interfaces réseaux dans les profils automatiquement. Cependant cela ne fonctionnera que si le nouveau système a une interface réseau avec exactement le même nom (dans la première capture d'écran c'est "LAN").

Définition des paramètres

Obtention des paramètres actuels

Pour charger les paramètres actuels dans le profil, il vous suffit de cliquer sur le symbole à côté du champ de saisie.
Pour obtenir tous les paramètres actuels clic droit sur le profil et choisissez "Obtenir les paramètres actuels".

Effacer les paramètres

Pour effacer une configuration : clic-droit et choisissez "Effacer".

Pour effacer les valeurs d'un champ IP il suffit de double-cliquer dessus

Pour effacer tous les paramètres du (sous)profil courant : clic-droit sur l'onglet du profil et choisissez «Effacer les paramètres».

Copier et coller des paramètres

Pour transférer des paramètres entre différents profils,vous pouvez copier et coller un profil par un clic-droit sur le profil d'origine en choisissant "Copier les paramètres, puis clic-droit sur le nouveau profil en choisissant "Coller les paramètres".

Pour copier et coller un paramètre spécifique : un clic droit dessus et choisissez "Copier" ou "Coller".

Activation du profil

Pour appliquer tous les paramètres du profil actuel, cliquez sur "Activer" ou si le programme est en cours d'exécution et minimisé sélectionnez le profil dans la barre de notification .
Notez que seuls les paramètres avec une case cochée seront activés.

La boîte de dialogue Activation contient des informations sur l'état d'avancement actuel de l'activation.
Là, vous pouvez voir les paramètres qui sont entrain d'être appliqués et si le processus est réussi ou non. Vous pouvez annuler le processus d'activation en cliquant sur le bouton "Annuler" à tout moment.

Toutes les modifications sont appliquées immédiatement. Toutefois, dans certains cas, certains paramètres prennent un peu de temps pour prendre pleinement effet ou nécessiter un redémarrage (en particulier "PC Name" et "Workgroup/Domaine").

La section Détails donne un aperçu de tous les paramètres configurés pour ce profil. Pour afficher les messages d'information et d'erreur complet ouvrir la section Journaux.

Pour trouver des solutions pour les erreurs et les problèmes les plus courants regarder la FAQ .


Résultats standards et leurs significations
Ok Tout a fonctionné comme prévu.
... Les réglages sont en cours d'activation.
- Paramètres ignorés sans essayer, parce qu'ils étaient incomplets ou erronés.
Erreur Dans la plupart des cas, les erreurs sont affichées sous forme de texte court en rouge comme "Fichier non trouvé". Dans le cas où vous recevez un code d'erreur, regarder le journal pour savoir si un message d'erreur existe pour ce code. Si vous n'avez aucune idée sur la raison de l'erreur, s'il vous plaît envoyer le code d'erreur et une description détaillée de l'erreur sur le forums de support .
Invalide Soit une valeur n'est pas valide, soit elle n'est pas spécifié ou non autorisés (caractères spéciaux, même workgroup que le nom de l'ordinateur, etc)
FF est actif Firefox est en cours d'exécution. Pour appliquer ces paramètres, vous devez d'abord fermer le navigateur.
Mode Compact & Mode Complet

Outre l'interface classique "Mode Complet", où vous pouvez modifier tous les paramètres d'un profil, une interface supplémentaire "Mode Compact» est disponible. Vous pouvez basculer entre ces modes à tout moment en cliquant sur le bouton correspondant dans le menu principal sur la droite.

Une fois que tous les profils sont mis en place, le mode compact est l'interface idéale. Vous avez un aperçu compact de tous les paramètres d'un profil sans se soucier de modification par erreur.

Dans le cas où vous utilisez l' administration NSM (avec "Modifier les paramètres de profil" décochée) le mode compact sera automatiquement utilisé pour empêcher l'utilisateur d'être en mesure de modifier les paramètres.

Barre de notification et Information de la zone de notification

Lorsque NetSetMan est en cours d'exécution, vous trouverez son icône dans la barre de notification (à côté de l'horloge) permettant l'activation rapide d'un profil.
Clic droit sur ​​l'icône pour ouvrir la zone de notification et choisir le profil à activer.

Si vous souhaitez exclure certains profils d'apparaître, vous pouvez les masquer individuellement. Pour ce faire, sélectionnez le profil dans la fenêtre principale, clic-droit sur l'onglet et cliquez sur "active en mode zone de notification et Compact". Le petit symbole à côté du texte montre que ce profil sera disponible dans la zone de notification. Cette option aura aussi un effet sur les profils présentés en mode compact. La visibilité du profil peut également être configuré dans le menu Gestion des profils .

Si vous laissez votre curseur sur l'icône NetSetMan dans la zone de notification le menu d'information apparaît. Il contient les informations IP actuellement utilisées par le système. Parce que les données sont recueillies en temps réel, il peut parfois y avoir un manque de réactivité. Pour réduire ce délai, vous devriez envisager de masquer des informations supplémentaires comme l'IP publique et les adresses MAC dans les Préférences.

Fonctionnalités
Aperçu

NetSetMan est conçu afin d'avoir un aperçu direct de tous les paramètres configurés dans un profil. Certaines des caractéristiques les plus importantes comme les réglages de base : IP, WiFi et l'imprimante par défaut peuvent être configurés directement depuis l'interface principale. D'autres paramètres plus complexes peuvent seulement être activés ici et s'afficheront de manière synthétiques. Pour les modifier il est nécessaire de les activer en premier, puis d'ouvrir la boîte de dialogue en cliquant sur le bouton [...].

Les sections suivantes vous donneront des instructions de base et des conseils utiles sur la façon d'utiliser toutes les fonctionnalités.

Etat de l'interface

Pour activer, désactiver ou redémarrer l'interface réseau sélectionnée, utilisez le bouton d'état de l'interface. La couleur par défaut va changer en fonction de l'état. En outre, vous pouvez voir l'état configuré dans l'info-bulle.

Si vous voulez activer une interface et désactiver une ou toutes les autres interfaces, vous devez créer des sous-profils pour chacun des interfaces. Vous pouvez le faire manuellement ou en utilisant l'élément de menu en bas du menu état des interfaces. Un sous-profil pour chaque interface réseau qui est inclus dans la liste des interfaces va être créer automatiquement et son statut sera positionné automatiquement. N'oubliez pas que vous pouvez personnaliser les interfaces qui sont inclus dans cette liste en utilisant la gestion des interfaces réseaux .

Réglages TCP/IP Avancées

Vous pouvez spécifier plusieurs adresses IP, passerelle, DNS et WINS par interfaces si vous avez besoin d'être sur plusieurs réseaux en même temps. Pour ce faire, cliquez sur le bouton "IP +» ou choisissez «Profil> Paramètres avancés TCP/IP" dans le menu principal.

Ici vous avez aussi la possibilité de gérer des paramètres plus avancés comme la table de routage, le suffixe DNS, NetBIOS et ipconfig.

  • Activer un groupe en cochant la case
  • Ajoutez une nouvelle valeur
  • Supprimer la valeur sélectionnée
  • Effacer toute la liste
  • Obtenez toutes les valeurs actuelles de ce paramètre à partir du système (cliquer sur le symbole dans la fenêtre principale pour lire la première valeur uniquement)
  • Réorganiser l'ordre par glisser-déposer
Connexions

Utilisez cette fonction pour connecter ou déconnecter tout type de connexions Windows, comme les réseaux d'accès à distance, haut débit mobile ou VPN. Le connexions compatibles sont énumérées dans Windows avec vos adaptateurs réseaux standard dans la boîte de dialogue Connexions réseau. D'un point de vue technique, toutes les connexions qui sont compatibles avec "RasDial" sont pris en charge.

Vous pouvez choisir de déconnecter toutes les connexions établies en cochant la case, ou pour connecter / déconnecter des connexions individuellement dans l'ordre.

Si votre connexion n'est pas présente, vous pouvez l'ajouter manuellement en choisissant "Ajout..." dans la liste de sélection et renseigner son nom tel qu'il apparaît dans Windows.

NetSetMan supporte plusieurs autres clients VPN comme OpenVPN, Shrew et Cisco. Si votre client VPN est compatible, NetSetMan listera les profils VPN dans la liste de sélection.

Hôtes

Dans des cas très spécifiques, il peut être nécessaire ou utile d'assigner manuellement des adresses IP aux noms ou des domaines. Notez que NetSetMan ne modifiera que ses propres entrées hôtes laissant les entrées déjà existantes intactes.

Lecteurs réseaux

Avec NetSetMan vous pouvez mapper et démapper un nombre illimité de lecteurs réseaux.
S'il vous plaît lisez les notes d'instructions au bas de la fenêtre en cliquant sur "[Show Aide]".

Lorsqu'un profil est activé et que la fonction "lecteur réseau" est sélectionné, NetSetMan comparera la liste de tous les lecteurs mappés actuellement avec la liste des lecteurs à mapper. Tous les lecteurs mappés qui sont dans les deux listes (même lettre de lecteur et même chemin d'accès) resteront intacts. Toutes les autres unités du réseau seront déconnectées avant les nouvelles ne soient connectées.

Scripts & Programmes

NetSetMan vous offre un moyen confortable pour gérer vos fichiers de script. Cependant, il n'a pas de langage de script propre. Cela signifie que vous ne pouvez utiliser que les langages de script que votre système d'exploitation est capable d'exécuter. Les langages de script courantes sont VBS, JS et les fichiers batch en ligne de commande. Utilisez votre moteur de recherche préféré pour trouver plus d'informations sur les scripts et ce que vous pouvez faire avec eux.

Ouvrir un fichier de script existant ou en créer un nouveau pour pouvoir éditer le script.
Vous pouvez avoir deux scripts qui seront exécutés avant et après la modification de tous les autres paramètres.
Si vous avez besoin d'un délai entre l'exécution du script et la modification des paramètres réseau, vous pouvez l'indiquer en bas.

Exécution du programme

NetSetMan offre, pour chaque profil, un moyen pratique d'exécuter des programmes externes.
Les programmes peuvent être lancés dans un contexte d'utilisateur limité (par défaut avec l'UAC activé) ou élevé avec des privilèges d'administrateur (sans déclencher une invite UAC). Des paramètres peuvent être ajoutés si nécessaire.
Les programmes peuvent être lancés normalement, maximisés ou minimisés.

Les processus en cours d'exécution peuvent être arrêtés en fonction de leur nom de fichier (indépendamment du lieu où ils se trouvent) ou de leur chemin d'accès exact.

L'exécution de chaque action peut être avancée (valeur négative) ou retardée (valeur positive).

SMTP / Email

Dans certains cas, l'adresse IP du serveur SMTP pour envoyer des emails peut varier en fonction de votre réseau. Vous pourriez avoir besoin d'une adresse IP locale comme 10.0.0.1, ou vous pourriez avoir besoin d'une adresse IP globale.

Au lieu d'essayer d'être compatible avec un grand nombre de différents clients de messagerie et de traiter avec différents profils d'utilisateurs NetSetMan résout ce problème avec une solution de contournement simple. Positionnez"nsm.smtp" comme adresse SMTP dans votre client de messagerie et configurer les adresses IP dans les profils en fonction. Lors de l'activation du profil, les valeurs seront définis en utilisant le fichier hosts de sorte que vous n'avez pas besoin de faire des modifications supplémentaires dans votre client de messagerie.

Paramètres Système

En plus des paramètres réseau habituelles, vous voudrez parfois modifier certains paramètres du système en fonction de votre position ou situation. Par exemple : à l'université, vous voudrez peut-être pour couper le volume sonore, changer le fond d'écran et activer le plan d'alimentation équilibrée.

Vous pouvez également utiliser la fenêtre Paramètres système pour changer rapidement quelques réglages qui, autrement, vous prendrait plus de temps à trouver dans les paramètres Windows. Il vous suffit de cliquer sur le bouton pour activer directement ce paramètre particulier.

Les paramètres pris en charge sont les suivants: le mode d'affichage, papier peint, couleur de bureau, écran de veille, modèle d'apparence, volume des haut-parleurs et du microphone, Gestion du son, pare-feu (incl. les exceptions et les notifications), Gestion de l'alimentation, fuseau horaire et adresse MAC.

WiFi

Un profil NetSetMan peut être utilisé pour se connecter à un profil WiFi existant. Pour ce faire, sélectionnez une interface WiFi dans la liste déroulante des interfaces réseau, puis sélectionnez un profil Wi-Fi dans la liste déroulante WiFi. Si vous voulez déconnecter votre connexion WiFi, choisissez l'entrée vide.

Au delà de la gestion WiFi de base, NetSetMan offre une gestion avancée du WiFi. Cliquez sur l'icône Wi-Fi ou utiliser le menu principal (Outils> NSM WiFi Management) pour y accéder. Vous trouverez une liste de tous les réseaux WiFi accessibles. Un fond jaune indique que le profil avec le même SSID existe déjà. Un réseau avec un fond vert est actuellement connecté. Vous pouvez vous connecter à un réseau en double-cliquant dessus ou en utilisant le Bouton "Connexion". S'il n'y a pas de profil existant pour cette connexion, vous pouvez choisir d'en créer un. Les paramètres (authentification, chiffrement, ...) seront pré-configurés automatiquement en fonction du réseau reconnu. Cependant, vous pourriez avoir besoin de les ajuster si, pour une raison quelconque, la reconnaissance automatique ne fonctionnait pas correctement .

NetSetMan travaille avec les profils natifs WiFi de Windows . Cela signifie que vous avez toujours accès aux mêmes profils que ceux utilisés dans Windows ou un autre logiciel de WiFi. Vous pouvez créer, modifier ou supprimer des profils WiFi.

Connexion par adresse MAC

Windows se connecte habituellement sur les réseaux sans fil sur la base de leurs SSID. Si plusieurs points d'accès partagent les mêmes groupes de SSID, Windows vous connecte à celui qui a le meilleur signal. Cependant, vous voudriez parfois, vous connecter à un point d'accès spécifique. Avec NetSetMan vous pourrez le faire !

Le logiciel de gestion du WiFi NSM affiche tous les réseaux et points d'accès disponibles sans les regrouper, afin que vous poussiez établir exactement la connexion que vous voulez. les infobulles vous montrent des informations supplémentaires comme l'adresse MAC pour chaque réseau. Le menu contextuel via clic droit vous permet de facilement copier une adresse MAC ou SSID.

Si vous souhaitez vous connecter à un réseau sans fil spécifique lors de l'activation d'un profil NetSetMan, vous pouvez le faire en ajoutant son adresse MAC à la sélection de réseau WiFi. Choisissez "Ajouter ..." et entrez l'adresse MAC de votre choix. Lors de l'activation du profil, NetSetMan se connectera au point d'accès correspondant. Attention, vous aurez toujours besoin d'avoir un profil WiFi approprié configuré sur votre système.

Exigences

NetSetMan utilise l'API WiFi natif de Windows. Aucun nouveaux drivers n'est nécessaire (limitant ainsi le risque de conflits). Les fonctions WiFi de NetSetMan ne fonctionne que sur les systèmes Windows qui prennent déjà en charge le WiFi. Heureusement, la plupart des versions utilisées de nos jours le gère. Si vous utilisez Vista ou une version ultérieure, c'est bon. Pour XP, vous devez avoir installé le SP2 ou le SP3.

Assurez-vous que le service de configuration automatique WLAN (Wlansvc) (Vista et versions ultérieures) ou de Configuration automatique sans fil (WZCSVC) (XP) est en cours d'exécution.

Si votre carte WiFi est livrée avec un logiciel propriétaire pour gérer les paramètres WiFi, il bloquera l'accès Windows dans la plupart des cas. Dans ce cas, vous aurez besoin de trouver la possibilité de passer à la gestion de Windows dans ce logiciel (si possible).
Parce qu'il y a beaucoup de fabricants différents avec leur propre logiciel, il n'y a pas de manière universelle pour y parvenir. Voici un exemple: Si vous utilisez un équipement FRITZ AVM!WLAN USB cliquez sur l'icône de la barre de son logiciel et choisissez "Windows WLAN".

Fonctions Avancées
AutoSwitch

Il est assez facile d'activer un profil en un seul clic à partir du menu de la barre de notification . Mais dans certains cas, il peut être encore plus facile d'activer des profils de manière entièrement automatique en fonction des certaines conditions. La fonction autoswitch intégrée fait exactement cela.

Après l'activation de la fonction AutoSwitch les conditions suivantes peuvent être configurées individuellement pour tous les profils :

Réseaux WiFi accessibles
  • Réseau sans fil (SSID ou MAC)
  • La puissance du signal du réseau sans fil inférieur / supérieur à une certaine valeur
Emplacements réseau
  • Fenêtres NLM: profils réseau générés automatiquement par Windows (Vista et versions ultérieures)
  • adresse MAC de la passerelle: attribué automatiquement
  • Ping: Disponibilité d'un certain serveur ou système dans le réseau
Statut de l'interface
  • Etat de la connexion de l'interface réseau local (connecté / déconnecté)
Calendrier
  • Certains jours de la semaine et intervalle de temps (par exemple Sa + Di, 08:00-18:00)

Bien qu'il n'y ait pas de limitation technique pour la quantité de conditions par profil, essayer de le garder aussi simple que possible. Dans la plupart des cas, il est suffisant de mettre une ou deux conditions pour obtenir le résultat souhaité.

Vous pouvez spécifier que tout ou au moins une des conditions doivent être remplies pour activer ce profil.

Remarque: Les Profils avec ID inférieures ont une priorité plus élevée dans le cas ou des conditions pour plusieurs profils sont remplies en même temps.

Par défaut, le processus d'activation est déclenchée par un message dans le coin en bas à droite avec un compte à rebours personnalisable, vous permettant de façon facultative d'interrompre le processus d'activation. Alternativement, une notification natif de Windows peut être utilisé ou le profil peut être activé sans aucun message en arrière-plan. Dans ce dernier cas, l'icône de la barre sera animée pendant le processus d'activation et sa couleur changera temporairement selon le résultat de l'activation afin de vous informer visuellement sur le processus automatisé.

Pour économiser les ressources système NetSetMan ne comparent pas les paramètres actuels aux paramètres du profil devant être activé. Cela signifie que si vous modifiez vos paramètres manuellement, NetSetMan ne remarque pas le changement.

La fonction AutoSwitch a quelques limitations dans XP: Windows NLM n'est pas disponible, les modifications dans les réseaux WiFi environnants peut être reconnue avec un retard notable.

Annotations

Vous pouvez créer de courtes notes pour chaque profil si vous avez besoin de vous rappeler ou d'indiquer le but d'un profil. Si une note a été créé pour un profil, son icône sera colorée dans la barre de menu. La note est alors visible en mode Complet comme un indice de l'icône et en mode compact au sommet des paramètres de profil.

Service NSM / compte utilisateur sans pouvoirs

NetSetMan nécessite des privilèges d'administrateur pour pouvoir modifier les paramètres réseau et système pour vous. Normalement, l'exécution d'une telle demande n'est pas possible pour un compte d'utilisateur sans pouvoirs. Les versions plus récentes de Windows limitent l'utilisation des compte administrateur avec l'UAC (User Account Control) et affiche un message à chaque démarrage du programme.

Pour résoudre ce problème NetSetMan comprend un composant spécial appelé service NSM qui fournit à l'application principale les privilèges requis. Le service est installé automatiquement mais peut également être installé/désinstallé manuellement dans les préférences (privilèges système> Activer / Désactiver).

Le service installé est visible dans la boîte de dialogue Services Windows (services.msc). En outre, il peut être installé / désinstallé ou démarré / arrêté en utilisant la ligne de commande :
nsmservice.exe /install /silent
nsmservice.exe /uninstall /silent
net start nsmService
net stop nsmService

Si vous obtenez l'erreur "signature non valide", assurez-vous que l'exe du programme n'a pas été modifié et que la stratégie de groupe Windows "Désactiver mise à jour automatique des certificats racine" est désactivée.

Si vous exécutez NetSetMan avec un compte d'utilisateur sans pouvoir où si le service NSM n'est pas disponible, vous recevrez un message expliquant le problème. Comme alternative, vous pouvez renseigner les information pour le compte administrateur à partir d'un compte local que NetSetMan chiffrera dans son fichier de configuration et utilisera pour se lancer avec les privilèges requis. Notez que dans ce cas, l'application fonctionnera dans le contexte de l'utilisateur dont les pouvoirs auront été utilisés.

Vérifiez la FAQ pour des informations supplémentaires sur cette fonction.

Changement de profil avant ouverture de session

Une caractéristique tout à fait unique et puissante est la possibilité d'activer vos profils sur l'écran d'ouverture de session. Cela permet la préparation paramètres réseau que vous avez besoin pour votre compte en activant un profil NetSetMan avant même la connexion.

Pour utiliser cette fonction service NSM doit être installé et l'option «Changement de Profil à l'écran d'ouverture de session Windows" (Préférences> Privilèges système) doit être activée.

Notez que cette fonctionnalité ne permet de modifier les paramètres qui sont partagés entre tous les utilisateurs. les paramètres spécifiques de l'utilisateur ne peuvent pas être modifiés en dehors d'une session utilisateur.

Administration NSM

Si vous configurez NetSetMan pour être utilisé par quelqu'un d'autre, vous devrez spécifier les actions que l'autre personne est autorisée à effectuer. Avec l'administration NSM (Préférences> Administration), vous pouvez faire exactement cela.

Lorsque vous activez ce mode, vous devrez entrer un mot de passe de 6 caractères. C'est seulement avec ce mot de passe que vous serez en mesure de modifier les paramètres d'administration NSM ou d'effectuer les actions que vous avez choisi de sécuriser (lancer NetSetMan, activer des profils, modifier les paramètres de profil, modifier les paramètres du programme).

Vous pouvez même verrouiller certains profils afin que l'utilisateur ne puisse pas les modifier, mais qu'il puisse être capable de les activer. Vous pouvez restreindre l'accès à certains paramètres comme "Workgroup et de domaine" tout en permettant à l'utilisateur de configurer et d'utiliser tous les autres paramètres.

Un checksum de contrôle de votre mot de passe sera écrit dans le Registre. Assurez-vous un utilisateur normal n'a pas de privilèges d'écriture à HKLM\Software \NetSetMan (sur x64: HKLM\Software\Wow6432Node\NetSetMan). Si vous avez oublié votre mot de passe, vous devrez supprimer cette clé manuellement. Si votre fichier de configuration est chiffré, sans le mot de passe, personne ne sera en mesure d'afficher ou le modifier.

Activation en ligne de commande

Dans certains cas particuliers, vous voudrez peut-être modifier votre profil sans utiliser l'interface graphique de NetSetMan: modifier les profils avec des raccourcis, activer un profil au démarrage, l'inclure dans un script complexe, etc

Les paramètres suivants sont disponibles :

-a # Active le profil avec le numéro #
-as # Active le profil en mode silencieux (la fenêtre d'Activation se fermera automatiquement si aucune erreur n'est survenue)
-h / -hide Exécute NetSetMan minimisé
f / -full Exécute NetSetMan en mode Complet (par défaut)
-c / -compact Exécute NetSetMan en mode Compact
-m / -multi Exécute une deuxième instance de NetSetMan. Attention: Toutes les instances actives écrivent dans le même fichier de configuration!

Un raccourci pour activer le profil # 0 en mode silencieux pourrait ressembler à ceci:
"C:\Program Files (x86)\NetSetMan\netsetman.exe" -as 0

Fonctions complémentaires de la version Pro
Profil de Groupes

Avec NetSetMan Pro vous êtes en mesure de créer un nombre illimité de groupes de profils. Chaque groupe contient 6 profils.

Pour ajouter, supprimer, réorganiser ou renommer un groupe de profil utilisez la gestion de profil dans la barre de menu.

Pour naviguer entre tous les groupes de profils disponibles choisissez n'importe quel groupe dans la liste déroulante dans la barre de menu.

Si vous avez plus d'un groupe, vos profils seront affichés regroupés dans le menu de la barre.

Navigateur / Proxy

Avec NetSetMan Pro vous permet de gérer les paramètres de proxy de Windows / Internet Explorer, Firefox et Opera (<= v12). Chrome et les versions ultérieures de l'Opéra ne prennent pas en charge les paramètres de proxy propres mais utilisent automatiquement les paramètres du système.

Tous les paramètres sont destinés à imiter les interfaces originales pour permettre les mêmes configurations que dans les navigateurs. En outre, vous pouvez modifier la page d'accueil et dans le cas où vous utilisez plusieurs profils dans Firefox ou Opera, vous serez également en mesure de choisir le nom du profil.

S'il vous plaît noter que Firefox doit être fermé avant de modifier ses paramètres.
Si vous avez de multiples profils Firefox et que le dossier de profil a changé (par exemple à cause d'un nouveau système), vous aurez besoin de re-sélectionner le profil dans la liste afin de mettre à jour votre fichier de paramètres.

Domaine

Pour rejoindre un domaine, entrez son nom, votre nom d'utilisateur et votre mot de passe. Votre nom d'utilisateur et mot de passe seront stockés chiffrés dans le fichier ini.

Habituellement il suffit d'entrer le nom d'utilisateur simplement. Toutefois, si vous obtenez une erreur, vous pouvez essayer d'entrer votre nom en suivant l'une des deux normes suivantes :
a) NETBIOS: DOMAIN\utilisateur
b) UPN: user@domain.local

Dans les domaines NT, un utilisateur ne peut être membre que d'un domaine à la fois. Lorsque vous rejoignez un autre domaine ou un workgroup votre adhésion dans le domaine précédente est perdue jusqu'à ce que vous re-basculiez sur le domaine précédent.

NetSetMan ne peut rejoindre un domaine que si votre compte d'utilisateur de domaine a la permission de le faire.
Ce privilège est appelé "Ajouter une Workstation à un domaine" et il doit être activé pour votre compte. Si vous ne disposez pas de ce privilège, s'il vous plaît demander le à votre administrateur.

Lire la FAQ pour des informations supplémentaires sur cette fonction.
